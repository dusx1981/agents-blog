要理解“自回归”在大模型预测中的位置，需要先厘清一个关键区别：**单次前向传播（计算过程）**和**完整响应生成（循环过程）**。

大模型预测的“完整流程”分为 4 个宏观阶段，而**自回归（Autoregression）特指第 3 阶段中的“循环解码”机制**，它贯穿了整个生成过程的每一步。

---

### 大模型预测的完整流程（4 个阶段）

#### 阶段 1：预处理与向量化（输入映射）
- **做了什么**：将你的提示词（Prompt）拆分成 Token（词元），并映射为初始向量，同时叠加位置编码（Positional Encoding）。
- **特点**：**一次性完成**，不涉及循环。

#### 阶段 2：上下文编码（Prefill / 预填充阶段）
- **做了什么**：模型将所有输入的 Token 向量并行“塞”进神经网络（数万亿次的矩阵乘法），进行全量注意力计算，生成并缓存 KV Cache（键值缓存）。这一步让模型“读懂”你的全部问题。
- **特点**：**并行计算**，只执行一次。这一步结束后，模型已经掌握了理解所需的全部上下文。

#### 阶段 3：循环解码生成（Decode / 自回归核心阶段）⭐
- **这就是自回归所处的阶段！** 这是一个 **“While”循环**，直到生成停止符（`<EOS>`）或达到最大长度。
- **循环体内的每一步（单次预测）**：
  1. **输入**：当前最新的 1 个 Token（第一步是阶段 2 产出的第一个新词，后续是上一步自己刚生成的词）。
  2. **前向计算**：利用阶段 2 缓存的 KV Cache（不再重算历史内容），仅计算新 Token 的注意力，得出原始得分（Logits）。
  3. **概率化（Softmax）**：对 Logits 应用 `Softmax`，转化为概率分布。
  4. **采样选词（Sampling）**：根据 `temperature` / `top-p` 等策略，掷骰子选定下一个 Token。
  5. **更新队列**：将选中的 Token 拼接到已有序列末尾，更新 KV Cache。
  6. **判断**：检查新 Token 是否为停止符，若是则跳出循环，否则回到步骤 1 继续。
- **核心特征**：**串行计算**（一个词一个词地蹦出来），后一个词的生成必须依赖前一个词的诞生。

#### 阶段 4：输出反向量化（后处理）
- **做了什么**：将生成的 Token ID 序列映射回人类语言文本（Detokenization），流式输出给你看。
- **特点**：**一次性完成**，不涉及循环。

```text
[用户输入] 
    ↓
[阶段 1 & 2] 预处理 + 并行编码 (Prefill)  → 耗时约 0.1秒 (一次性)
    ↓
╔═══════════════════════════════════════════════════════════╗
║  [阶段 3] 自回归解码循环 (Decode Loop)  ←  ⭐ 这里就是自回归 ║

║  第1步: 生成 Token_A (依赖输入)                        ║
║      ↓                                               ║
║  第2步: 生成 Token_B (依赖 输入+Token_A)              ║
║      ↓                                               ║
║  第3步: 生成 Token_C (依赖 输入+Token_A+Token_B)     ║
║      ↓                                               ║
║  ... 直到遇 <EOS> 跳出                               ║
╚═══════════════════════════════════════════════════════════╝
    ↓

```

### 关键结论（必记）

1. **位置锁定**：自回归 **只发生在阶段 3（解码/生成阶段）**。阶段 1、2、4 都不属于自回归。
2. **本质定义**：自回归在这里指的是 **“将上一次循环的输出，作为下一次循环的输入”** 这一循环机制。
3. **性能瓶颈**：因为自回归是串行的（生成第 100 个字必须等前 99 个），所以这是大模型推理中最耗时的部分。这也是为什么 DeepSeek 会引入 **MLA（多头潜在注意力）** 和 **MTP（多词元预测）** 来优化这一阶段的缓存大小和并行度。
