# Pi 提示模板参数用法详解与处理流程

提示模板提供了 `/name` 命令来将预定义的 Markdown 片段快速展开为完整的提示词，并通过位置参数实现动态内容替换。下面用具体示例补充说明参数的各种用法，并揭示从用户输入到模板展开的完整处理流程。

## 一、参数用法示例

### 1. 基础位置参数：`$1`, `$2`, ...

**模板文件** `.pi/prompts/component.md`：

```markdown
---
description: 生成一个 React 组件
---
请创建一个名为 `$1` 的 React 组件，放在 `src/components/$1.tsx`。
该组件需要包含以下功能：
$2
```

**用户调用**：

```
/component Button "点击计数、禁用状态、loading 样式"
```

**展开后编辑器中的内容**：

```
请创建一个名为 Button 的 React 组件，放在 src/components/Button.tsx。
该组件需要包含以下功能：
点击计数、禁用状态、loading 样式
```

> `$1` 被第一个参数 `Button` 替换，`$2` 被第二个参数（带引号的句子）替换。

### 2. 所有剩余参数合并：`$@` 或 `$ARGUMENTS`

**模板** `.pi/prompts/summarize.md`：

```markdown
---
description: 用指定语言总结项目
---
请使用 $1 对当前项目进行总结，重点包括：$@
```

**调用**：

```
/summarize 中文 项目结构 核心依赖 主要功能
```

**展开后**：

```
请使用 中文 对当前项目进行总结，重点包括：项目结构 核心依赖 主要功能
```

> `$1` 取 `中文`，`$@` 将第二个参数开始的全部内容（`项目结构 核心依赖 主要功能`）拼接在一起。

如果用户调用时只写 `/summarize 中文`，`$@` 将是空字符串，模板末尾会出现 “重点包括：”（后接空白），这是需要注意的边缘情况，可以通过在模板中做条件判断（例如先预留一个占位符）来优化，不过模板本身不支持条件逻辑，可以在设计时让参数本身变得可选。

### 3. 参数切片：`${@:N}` 和 `${@:N:L}`

- `${@:N}` – 从第 N 个参数开始到结尾
- `${@:N:L}` – 从第 N 个参数开始的 L 个参数

**模板** `.pi/prompts/story.md`：

```markdown
---
description: 写一个短故事
---
用${@:2}风格写一个关于 $1 的短故事。
要求：${@:3:2}
```

**调用**：

```
/story "一只猫" 诙谐 必须包含反转结局 字数不超过200字 采用第一人称
```

参数列表：
1. `一只猫`
2. `诙谐`
3. `必须包含反转结局`
4. `字数不超过200字`
5. `采用第一人称`

**展开后**：

```
用诙谐 字数不超过200字 采用第一人称风格写一个关于 一只猫 的短故事。
要求：必须包含反转结局 字数不超过200字
```

分析：
- `$1` → `一只猫`
- `${@:2}` → 从第 2 个参数开始到末尾：`诙谐 必须包含反转结局 字数不超过200字 采用第一人称`（直接拼接）
- `${@:3:2}` → 从第 3 个参数开始取 2 个：`必须包含反转结局 字数不超过200字`

> **提示**：切片计数是从 1 开始的，且参数是用空格或引号分隔的，所以如果某个参数本身含有空格，必须用引号包裹。

## 二、处理流程

当用户在 Pi 编辑器中输入 `/component Button "onClick handler"` 时，内部处理流程如下：

### 步骤 1：命令识别与解析

1. 用户按下 `/` 键，触发命令补全。
2. 自动补全会列出所有可用的命令，包括扩展命令、内置命令、模板（以 `prompts/` 目录下的 `.md` 文件名展示）、技能（`/skill:name`）。
3. 当用户选择模板名称（如 `component`）并补全后，编辑器中的文本变为 `/component `。

### 步骤 2：参数提取

1. 用户继续输入剩余部分（可以包含空格、引号参数）。
2. Pi 的输入解析器将 `/command args` 分割为命令名 `component` 和参数字符串。
3. 参数字符串按照**类 shell 的规则**进行分割：空格作为分隔符，但引号内的内容作为一个整体参数（支持双引号和单引号）。
   - 例如：`Button "onClick handler" disabled` → 三个参数：`Button`, `onClick handler`, `disabled`
4. 参数存入数组 `args = ["Button", "onClick handler", "disabled"]`。

### 步骤 3：模板加载与变量替换

1. Pi 根据命令名 `component` 在模板目录中查找 `component.md` 文件。
2. 读取文件内容，提取前置元数据（`description` 等）和主体模板文本。
3. 对模板文本进行变量替换：
   - 替换 `$1` 为 `args[0]`（若不存在则保留原样或替换为空字符串，取决于实现，通常是空字符串）。
   - 替换 `$2` 为 `args[1]`，以此类推。
   - 替换 `$@` 或 `$ARGUMENTS` 为所有参数以空格连接后的字符串（如果无参数则为空字符串）。
   - 处理切片 `${@:N}` 和 `${@:N:L}`，通过数组切片得到子数组后连接。
4. 未定义的变量（如模板中写了 `$3` 但用户只给了一个参数）将被替换为空字符串，模板不会因此报错。

### 步骤 4：注入编辑器

1. 替换后的完整文本被**直接插入到当前编辑器光标位置**，完全替代用户刚刚输入的 `/component ...` 命令文本。
2. 编辑器进入正常编辑模式，用户可以继续修改文本，然后按回车发送给模型。

整个过程是**同步且无网络请求**的，模板文件本地存在，无需模型参与，因此极快。

## 三、与技能和扩展的配合

模板的参数机制也可以和技能形成联动。例如，技能 `pdf-tools` 的 `SKILL.md` 中可以指导模型：**“在分析 PDF 前，请使用 `/analyze-pdf` 模板生成结构化分析提示”**。模型会自动发起 `/analyze-pdf` 命令（或者用户手动输入），模板展开后启动一次专门针对 PDF 分析的对话。

同样，扩展也可以注册命令来间接使用模板，或者通过 `pi.sendUserMessage` 动态展开模板内容并注入会话，实现更复杂的自动化流程。

## 四、总结

Pi 提示模板的参数系统做到了**简单却强大**：
- **足够简单**：只支持位置参数和切片，没有复杂的逻辑或循环，学习成本几乎为零。
- **足够表达力**：通过切片、引号、`$@` 等机制，可以构建丰富的动态提示词。
- **可组合**：模板文件就是 Markdown，方便版本控制和团队分享，是“提示工程”的最佳实践载体。

理解它的处理流程后，你可以更自信地在工作流中创建大量小而美的模板，将重复性的提示词输入固化为一条条 `/命令`，大幅提升与 AI 协作的效率。
